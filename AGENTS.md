# AGENTS.md

## Cursor Cloud specific instructions

### Project overview

DAS (Distributed Acoustic Sensing) Canyon Dashboard for SR-190 Big Cottonwood Canyon, Utah. Front-end-only prototype that simulates real-world DAS monitoring of a fiber optic cable along a canyon roadway. See `Scope/Scope.md` for the full design specification.

### Tech stack

- **Runtime**: Node.js 20 LTS
- **Build tool**: Vite (dev server + production build)
- **Map library**: MapLibre GL JS with 3D terrain (AWS Terrarium tiles, no API key)
- **Linting**: ESLint (flat config in `eslint.config.js`)
- **Testing**: Vitest (test files in `test/`)
- **Preprocessing**: Python 3 (no pip dependencies — uses only stdlib: `json`, `math`, `os`)
- **No backend / no database** — purely static frontend with simulated data

### Common commands

| Task | Command |
|------|---------|
| Dev server | `npm run dev` (serves on `http://localhost:5173`) |
| Lint | `npm run lint` |
| Tests | `npm run test` |
| Build | `npm run build` |
| Preview prod build | `npm run preview` |
| Generate sample data | `python3 scripts/generate_sample_data.py` |
| Preprocess GIS data | `python3 scripts/preprocess_fiber.py` |

### Data pipeline

1. Raw GIS files go in `data/raw/` (fiber.geojson, road.geojson, mileposts.geojson, crossings.geojson)
2. Run `python3 scripts/preprocess_fiber.py` to produce processed files in `data/`
3. The Vite dev server serves `data/` as publicDir — all JSON/GeoJSON files are available at `/filename`

### Gotchas

- The Vite dev server binds to `0.0.0.0:5173` so it is accessible from the Desktop pane browser.
- 3D terrain tiles come from AWS Terrarium (`s3.amazonaws.com/elevation-tiles-prod`) — no API key needed but requires network access.
- Map base tiles come from OpenStreetMap — also requires network access.
- The preprocessing script uses only Python stdlib (no geopandas/shapely). It handles segment stitching, channel generation, milepost interpolation, crossing detection, and side-of-road computation.
- `data/raw/` contains sample GeoJSON generated by `scripts/generate_sample_data.py`. These will be replaced by real GIS data later.
- When real GIS data is added to `data/raw/`, re-run `python3 scripts/preprocess_fiber.py` to regenerate the processed files.
